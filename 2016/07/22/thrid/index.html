Position

###Position

值 描述
absolute 生成绝对定位的元素，相对于static 定位以外的第一个父元素进行定位。元素的位置通过“left”，“top”，“right”以及“bottom”属性进行规定。
fixed 生成绝对定位的元素，相对于浏览器窗口进行定位。元素的位置通过“left”，“top”，“right”以及“bottom”属性进行规定。（ie6不支持）
relative 生成相对定位的元素，相对于其正常位置进行定位。如果top:10px;那么元素会相对于以前的正常位置向下移10,且不会影响其他元素
static 默认值，没有定位，元素出现在正常的流中。
inherit 规定应该从父元素继承position属性的值

 1.absolute(绝对定位)
 position属性为absolute的元素脱离了文本流，在文本中不再占有位置，可选取具有定位的父级对象或者body坐标原点进行定位，且元素可以用z-idenx进行层次分级。所以同一父元素下的所有absolute如果没有设置盒子属性的话，是根据父级对象的坐标作为始点的，左上角可能会会挤到同一位置。 

<style>
body{
    margin:0;   
    padding:0;  
    list-style:none;
}
.aside{
    position:absolute;
    width: 200px;
    height: 200px;
    /*margin-left: 90px;*/
    background-color: black;
    z-index: 2;
}
.bside{
    position: absolute;
    width: 300px;
    height: 300px;
    left: 20px;
    top:20px;
    /*margin-bottom: -100px;*/
    background-color: green;
    z-index: 1;
}
</style>
</head>
<body>
    <div class="aside"></div>
    <div class="bside"></div>
</body>

此刻body就是两个position为absolute的元素的父元素，left、top是相对于父元素来说的。

 2.relative（相对定位）
 position属性为relative的元素是相对于元素本身在文档中应该出现的位置来移动这个元素，实际上该元素依然占据文档中原有的位置，只是视觉上相对原来的位置有移动。重要的是，该元素不会影响其他元素的位置。

<style>
body{
    margin:0;   
    padding:0;  
    list-style:none;
}
.aside{
    position:fixed;
    width: 200px;
    height: 200px;
    left: 20px;
    top:110px;
    /*margin-left: 90px;*/
    background-color: black;
    z-index: 2;
}
.bside{
    /*position: absolute;*/
    width: 300px;
    height: 300px;
    /*margin-bottom: -100px;*/
    background-color: green;
    z-index: 1;
}
</style>
</head>
<body>
    <div class="aside"></div>
    <div class="bside"></div>
</body>

aside的位置只是相当于他以前的位置进行移动，但并没有影响到bside的位置。

 3.fixed（绝对定位）
 把position属性设置为fixed的元素的位置是对于浏览器的窗口而言的，始终处于窗口的某个位置。

 4.float的问题
 在网页设计时经常会用到浮动来对页面进行布局，但是浮动所带来的不确定因素却很多（例如：IE浏览器的兼容问题）。相对来说，在有些布局中定位使用会更加简单、快捷、兼容性更好（relative与absolute相结合来使用）。
